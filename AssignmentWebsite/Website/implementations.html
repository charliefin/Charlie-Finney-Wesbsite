Implementation used on the Website

Implementation Styles

On this page, I will be going through some of the implementation styles that I have used on this site, and the reasons for them. These will include Inline, Embedded & External implementation styles.

Inline

As explained on the previous webpage, the inline style implementation requires adding the style attribute to your chosen HTML element. This adds a unique style to just that element. This is certainly useful for wanting only certain things to have a unique style, or if you dont have access to .css files. Lets take a quick look at its advantages and disadvantages when compared to the other implementations...

Advantages

  • Quickly insert CSS rules - This implementation style allows you to easily insert CSS rules to HTML elements quickly. This can be very useful for testing changes, and making quick performance fixes.
  • No need for creating documents - You do not need to create and link a document for the styles to take affect. External implementations on the other hand do.

Disadvantages

With advantages, it comes its disadvantages. Here are some disadvantages about inline...

  • Becomes tedious and time consuming - Adding styles tags to every HTML element can become very time consuming. The structure of your page may become messy and difficult to read and navigate through.
  • Can slow download times - The more elements you style, the slower download times become. It also negatively effects the page size, increasing it.

Embedded

The embedded style implementation involves adding the "style" tag in the "head" tag. This implementation method is very useful for styling a page, as you can add multiple styles under the "style" tag related to the page. A problem with this though is that it can also be very time consuming when applying these styles to all pages, as it requires you to copy the styles over again and again with each page. Heres its advantages and disadvantages...

Advantages

  • ID selectors - With this implementation, you are able to make use of ID selectors. This allows the user to select a specific unique element. This is done by adding a hash (#) character, followed by the id of the element.
  • Won't need multiple files - As this will be under the same HTML file, you won't have to add multiple files.

Disadvantages

  • Slow down page loading times - The more styles you add, the slower the page's loading times get. It will also effect the page size, increasing it.

External

The External style implementation involves linking a .css file to the HTML page. This means that the user can add the page styling to one file, and link it across all the pages that will use the styles. This is very efficient for larger websites that have multiple pages. Its very efficient due to making one change the the .css file, it can apply the changes to all pages across the website. Here are some advantages and disadvantages...

Advantages

  • Cleaner structure - As all the page styling is contained on one seperate file, it saves lots of repeated HTML and CSS across all pages, tidying the structure, and decreasing page size.
  • Using the same file - You do not need to use multiple .css files. You can just make use of one across all pages.

Disadvantages

  • Rendering pages - The pages may take longer to fully render as they have to wait for the .css file to load.
  • Using multiple .css files - The more .css files you use, the higher download times will increase.

Conclusion

In my opinion, I find both External and Embedded the most useful. They both work together, with External saving lots of space and giving a cleaner structure, and embedded giving easy to read accessibility to the style of the page. I think that inline implementations will hardly be used unless you want a very specific HTML element needing a different style to the rest. I have only made use out of this a couple of times, whereas ive seen much more use in External and Embedded implementations.
